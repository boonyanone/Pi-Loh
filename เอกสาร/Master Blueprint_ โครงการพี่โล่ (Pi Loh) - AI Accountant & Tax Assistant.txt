🛡️ Master Blueprint: โครงการ "พี่โล่" (Pi Loh)
ฉบับสมบูรณ์สำหรับการพัฒนาด้วย Antigravity
1. วิสัยทัศน์และเป้าหมาย (Vision & Mission)
"พี่โล่" คือ AI Employee ส่วนตัวสำหรับ SME ไทย ที่เปลี่ยนเรื่องบัญชีและภาษีที่ยุ่งยากให้เป็นเรื่องง่ายและปลอดภัย โดยยึดหลัก "Privacy by Design" ข้อมูลทางการเงินทั้งหมดจะถูกประมวลผลภายในเครื่องของผู้ใช้ (Local-first) ไม่มีการส่งข้อมูลดิบขึ้น Cloud
2. สถาปัตยกรรมทางเทคนิค (Technical Architecture)
นี่คือส่วนที่สำคัญที่สุดสำหรับการสื่อสารกับระบบ Antigravity เพื่อให้เข้าใจโครงสร้างเชิงเทคนิคที่ซับซ้อน:
2.1 โครงสร้างหลัก (System Stack)
* Frontend: React.js + Tailwind CSS (เน้น UI ที่สะอาดตา ขอบมนสไตล์ Modern Enterprise)
* Backend Wrapper: Tauri (Rust) - ใช้สำหรับควบคุมความปลอดภัย เข้าถึงไฟล์ในเครื่อง และจัดการ Lifecycle ของแอป
* AI Inference Engine: ZeroClaw (รันแบบ Sidecar Process) เพื่อให้โปรแกรมสามารถรัน LLM ได้โดยไม่ต้องติดตั้ง Python หรือ Library ภายนอกในเครื่องลูกค้า
* Local LLM: โมเดลภาษาไทยเฉพาะทาง (เช่น Typhoon 2.1 หรือ Llama-3-Thai) ในรูปแบบ GGUF (4-bit/8-bit quantization) เพื่อให้รันบน PC ทั่วไปได้
2.2 ระบบความรู้ (Knowledge Engine - RAG)
* Retrieval-Augmented Generation (RAG): ระบบจะทำการ "เปิดตำราตอบ" โดยดึงข้อมูลจากฐานข้อมูล Vector ในเครื่อง (เช่น LanceDB หรือ SQLite-vss)
* Data Sources: ประมวลรัษฎากรล่าสุด, มาตรฐานบัญชีสำหรับ NPAEs, คู่มือการเสียภาษีของกรมสรรพากร
3. รูปแบบและการแจกจ่ายโปรแกรม (Distribution & Build)
* Development Platform: พัฒนาและทดสอบบน Mac (ใช้ Antigravity ในการโค้ด)
* Target Platform: ผลลัพธ์สุดท้ายคือ .exe สำหรับ Windows (แจกจ่ายแบบ Standalone Download)
* Build Pipeline: ใช้ GitHub Actions เพื่อ Compile โค้ดจาก Mac ให้กลายเป็นไฟล์ Windows .exe
* Installation Strategy: Smart Installer
   * ตัวติดตั้งหลักจะมีขนาดเล็ก (50-100MB)
   * เมื่อรันครั้งแรก โปรแกรมจะทำการดาวน์โหลดโมเดล LLM เฉพาะทาง (Knowledge Package) มาติดตั้งใน AppData ของผู้ใช้โดยอัตโนมัติ
4. ฟีเจอร์หลัก 3 ระยะ (Product Pillars)
ระยะที่ 1: ที่ปรึกษา AI (AI Consultant) - Focus Now
* แชทบอทอัจฉริยะที่ตอบคำถามภาษีและบัญชีไทย
* ต้องอ้างอิงมาตรากฎหมายทุกครั้งที่ตอบ
* มีปุ่มทางลัด (Presets) สำหรับคำถามที่ SME พบบ่อย (เช่น ค่าน้ำมัน, ค่ารับรอง, VAT ต้องห้าม)
ระยะที่ 2: เครื่องยนต์ตรวจสอบ (Audit Engine)
* ระบบ Drag & Drop ไฟล์ Excel บัญชี
* AI ทำการ Audit ค้นหาจุดผิดปกติ (Anomalies) และความเสี่ยงทางภาษี
* แปลงข้อมูลเข้าสู่มาตรฐานกลางเพื่อให้พร้อมสำหรับการปิดงบ
ระยะที่ 3: ตลาดกลาง (Professional Marketplace)
* ระบบประมูลงาน (Bidding) เชื่อมต่อเจ้าของธุรกิจกับผู้สอบบัญชี (CPA/TA)
* ส่งออกข้อมูลที่ AI ตรวจสอบแล้วให้ผู้สอบบัญชี เพื่อลดเวลาและค่าใช้จ่ายในการปิดงบ
5. โมเดลการสร้างรายได้ (Monetization Model)
1. Knowledge Package Updates: ขายชุดข้อมูลกฎหมายอัปเดตใหม่ๆ (เช่น ภาษีปี 2569) ในราคา Micro-payment (หลักร้อยบาท)
2. Marketplace Transaction Fee: หักค่าธรรมเนียมจากการจบงานในระบบประมูลผู้สอบบัญชี
3. Premium Dashboards: ระบบวิเคราะห์สุขภาพธุรกิจเชิงลึกสำหรับเจ้าของกิจการ (Subscription)
6. แนวทาง UI/UX (Design Guidelines)
* Trust Indicators: มีสถานะ "Safe Pulse" (ไฟกระพริบสีเขียว) และข้อความ "Local-only" เพื่อให้ผู้ใช้มั่นใจ
* Simplified Complexity: ลดการใช้ศัพท์บัญชียากๆ (Jargon) แต่ใช้ Visual อย่าง Gauge หรือ Health Score แทน
* Contextual Reference: หน้าจอแชทต้องมีแถบด้านข้างที่แสดงมาตรากฎหมายที่ AI กำลังอ้างอิงอยู่แบบ Real-time
7. แผนปฏิบัติการเบื้องต้น (Initial Hand-off for Antigravity)
1. Initialize: เริ่มต้นโปรเจกต์ Tauri + React บน Mac
2. Sidecar Setup: เตรียมระบบเรียกใช้ ZeroClaw Binary ผ่าน Rust
3. RAG Implementation: พัฒนาระบบค้นหาข้อมูลจากมาตรากฎหมาย (ประมวลรัษฎากร)
4. UI Migration: ย้ายองค์ประกอบจาก pi_loh_app.html เข้าสู่ระบบ Component ของ React
