🛡️ พิมพ์เขียวฉบับสมบูรณ์: โครงการ "พี่โล่" (Pi Loh)
Master Blueprint for Development on Antigravity (Mac for Windows Target)
1. วิสัยทัศน์และคอนเซปต์ (Core Vision & Concept)
* ชื่อโครงการ: พี่โล่ (Pi Loh)
* นิยาม: AI Employee ส่วนตัวสำหรับ SME ไทยที่เป็นทั้งที่ปรึกษาภาษีและพนักงานบัญชีอัจฉริยะ
* จุดขายหลัก (USP): "Privacy by Design - Local-First AI" ข้อมูลทางการเงินและความลับธุรกิจประมวลผลในเครื่องผู้ใช้ 100% ไม่ส่งขึ้น Cloud ตอบโจทย์ความกังวลของ SME ไทย
2. โครงสร้างเชิงเทคนิคขั้นสูง (Technical Stack & Architecture)
2.1 กลยุทธ์การพัฒนา (Cross-Platform Strategy)
* เครื่องมือพัฒนา: พัฒนาและทดสอบบน Mac (ผ่าน Antigravity)
* เป้าหมายการแจกจ่าย: Windows Executable (.exe) แบบ Standalone (ไม่ต้องผ่าน Store)
* การ Build: ใช้ GitHub Actions เพื่อ Compile โค้ดจาก Mac ให้เป็นไฟล์ Windows .exe
2.2 โครงสร้างโปรแกรม (System Architecture)
* Framework: Tauri (Rust) - เป็น Wrapper หลักที่เบา ปลอดภัย และจัดการระบบไฟล์ในเครื่องได้ดี
* UI Framework: React.js + Tailwind CSS (อิงตามไฟล์ pi_loh_app.html)
* AI Inference Engine: ZeroClaw (รันแบบ Sidecar Process)
   * ทำหน้าที่เป็นตัวประมวลผล AI โดยไม่ต้องให้ผู้ใช้ติดตั้ง Python หรือ Library ภายนอก
   * เชื่อมต่อกับ UI ผ่าน Rust Commands ใน Tauri
* Knowledge Retrieval (RAG):
   * Vector Database: LanceDB (Serverless, เก็บเป็นไฟล์เดียวในเครื่องลูกค้า)
   * Embedding Model: BGE-M3 (เก่งภาษาไทยและ Multilingual สำหรับค้นหากฎหมาย)
2.3 โมเดลภาษา (The Brain - LLM)
* Model: Typhoon 1.5 / 2.1 (7B/8B) ในรูปแบบ GGUF (4-bit/8-bit Quantized)
* ความเฉพาะทาง: เน้นความเข้าใจบริบทกฎหมายภาษีไทย มาตราประมวลรัษฎากร และมาตรฐานบัญชีไทย (TFRS for NPAEs)
3. แผนการดำเนินงานและฟีเจอร์ (Product Roadmap)
Phase 1: ที่ปรึกษา AI (AI Consultant) - เป้าหมายปัจจุบัน
* Feature: แชทบอทถาม-ตอบปัญหาภาษีและบัญชีไทย
* Intelligence: ระบบ RAG ที่ "เปิดตำราตอบ" โดยอ้างอิงมาตรากฎหมายทุกครั้ง
* Installation: Smart Installer (ตัวติดตั้ง .exe ขนาดเล็ก -> โหลด LLM Package เฉพาะทางเพิ่มเมื่อเปิดแอปครั้งแรก)
Phase 2: เครื่องยนต์ตรวจสอบ (Audit Engine)
* Feature: ระบบตรวจสอบไฟล์ Excel บัญชีเพื่อหาจุดผิดปกติ (Anomalies) เช่น ภาษีซื้อต้องห้าม หรือยอด VAT ไม่ตรง
Phase 3: ตลาดกลาง (Marketplace)
* Feature: เชื่อมต่อ SME กับผู้สอบบัญชี (CPA/TA) ผ่านระบบประมูลงานที่เตรียมข้อมูลไว้พร้อมปิดงบ
4. แนวทางการออกแบบ UX/UI (Design Principles)
* Visual Trust & Professionalism: ใช้โทนสีที่สื่อถึงความมั่นคงและมืออาชีพ (Professional Palette) ควบคู่กับการใช้พื้นที่ว่าง (Whitespace) อย่างเหมาะสม เพื่อลดความตึงเครียดของข้อมูลบัญชีที่ซับซ้อน
* Human-Centric Interaction: เน้นองค์ประกอบที่ดูเป็นมิตรและเข้าถึงง่าย (Intuitive Design) โดยใช้รูปทรงโค้งมนและเลย์เอาต์ที่สะอาดตา เพื่อสร้างประสบการณ์การใช้งานที่ลื่นไหลและลดภาระทางความคิด (Cognitive Load) ของผู้ใช้
* Real-time System Feedback: แสดงสถานะการทำงานและระดับความปลอดภัยอย่างชัดเจน (Visual Feedback) เช่น ระบบ "Safety Pulse" เพื่อยืนยันว่าข้อมูลกำลังถูกประมวลผลอย่างเป็นส่วนตัวภายในเครื่อง
* Hierarchy & Readability: ให้ความสำคัญกับลำดับขั้นของข้อมูล (Visual Hierarchy) และความอ่านง่ายของตัวอักษร (Typography) โดยเฉพาะในการแสดงผลข้อกฎหมายและตัวเลขทางการเงิน เพื่อป้องกันความผิดพลาดในการตีความ
* Contextual Support: ออกแบบส่วนช่วยเหลือที่สอดคล้องกับบริบทการใช้งาน เช่น แถบ "Reference Panel" ที่แสดงข้อมูลอ้างอิงทันทีเมื่อ AI ตอบคำถาม เพื่อสร้างความน่าเชื่อถือในทุกขั้นตอน
5. โมเดลการสร้างรายได้ (Monetization Strategy)
1. Knowledge Package Updates: ขายชุดความรู้กฎหมายภาษีใหม่ๆ (Micro-payment) เมื่อมีการเปลี่ยนแปลงกฎหมายรายปี
2. Marketplace Commission: หักค่าธรรมเนียมจากการจบงานในระบบประมูลผู้สอบบัญชี
3. Premium Analytical Tools: ระบบแดชบอร์ดวิเคราะห์งบการเงินขั้นสูง
6. แผนการดำเนินการทันทีสำหรับ Antigravity (Implementation Steps)
1. Environment Setup: ตั้งค่าสภาพแวดล้อม Tauri บน Mac พร้อมรองรับ Sidecar Binary
2. ZeroClaw Integration: นำ ZeroClaw Binary (Mac/Windows) วางในโฟลเดอร์ src-tauri/binaries
3. UI Migration: ย้ายโค้ดจาก pi_loh_app.html เข้าสู่ React Component
4. RAG Logic: เขียนสคริปต์ย่อยกฎหมาย (Chunking) และทำ Embedding ของประมวลรัษฎากรชุดแรก
5. Build Pipeline: ตั้งค่า Workflow สำหรับส่งโค้ดไป Build เป็น .exe ผ่าน GitHub Actions
7. ข้อมูลสำคัญในการติดตั้ง (Specific Requirements)
* Windows Security: ไม่ใช้ใบรับรองราคาสูงในช่วงแรก เน้นการสอนผู้ใช้กด "More info > Run anyway" ตามรูปแบบโปรแกรมบัญชีไทยทั่วไป
* Model Management: แอปต้องมีระบบดาวน์โหลดไฟล์ .gguf จาก Server ของเราไปเก็บที่ AppData/Local/PiLoh โดยอัตโนมัติ
